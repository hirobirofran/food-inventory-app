# 食材管理アプリ — Claude への申し送り

## このプロジェクトは何か

自炊をラクにする食材管理 PWA。「今何がある？何が作れる？何を買えばいい？」をAIが答えてくれるアプリ。

オーナー: ひろゆきさん（個人利用。将来的に家族共有あり）

## 技術スタック

- **フロントエンド**: Next.js 15 (App Router) + TypeScript + Tailwind CSS
- **データ**: Google Sheets API（データベース代わり）
- **AI**: Claude API `claude-sonnet-4-6`（レシピ提案・OCR）
- **デプロイ**: Vercel
- **PWA**: スマホのホーム画面に追加して使う

## 現在の状態

- Phase 1 UI 実装済み（モックデータで動作）
- Google Sheets 連携・Claude API 連携は**未実装**
- 次にやること → `docs/TASKS.md` の「次にやること」セクションを参照

## 重要ルール

- `.env.local` と Google サービスアカウント JSON キーは**絶対にコミットしない**
- `docs/` フォルダに詳細ドキュメントがある（人間向け）
- 各セッション終了時に `docs/TASKS.md` と `docs/KNOWLEDGE.md` を更新する

## 秘密情報・識別子の扱い（重要）

- **秘密情報**（パスワード・トークン・API キー・サービスアカウント private_key）は Claude のチャットに絶対に貼らない／出力させない。ソースコードにハードコードもしない
- **環境固有の識別子**（スプレッドシート ID、Vercel プロジェクト ID、Google Cloud プロジェクト名など）も、秘密情報でなくても**リポジトリに含めない**。`.env.local` 等に置き、`.env.local.example` ではプレースホルダで示す
- 実行に必要な識別子はすべて環境変数で渡す。スクリプト内でハードコードしない（`scripts/seed-demo.ts` のように本番/デモを取り違えない安全チェックは、env var の比較で実現する）
- Vercel の環境変数はダッシュボード UI から直接入力する（CLI stdin 経由はシェル履歴に残る）
